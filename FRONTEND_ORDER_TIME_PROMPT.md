# Работа со временем заказа на фронтенде

## Что нужно внедрить

### 1. Отображение времени выноса заказа
- Если `scheduledAt` не указан при создании, бэкенд автоматически установит текущее время + 1 час
- Отображать время выноса в формате: "Вынос в 17:30" (пример: если сейчас 16:30, показать 17:30)
- Использовать `scheduledAt` из ответа API после создания заказа

### 2. Выбор времени при создании заказа
- Добавить поле для выбора времени доставки (datetime picker)
- Поле опциональное - можно не указывать
- Диапазон выбора: от текущего времени до +1 час
- Формат отправки: ISO 8601 (например: "2024-01-17T11:00:00Z")
- Можно указать любое время, даже если оно в прошлом (например, 11:00 при текущем времени 16:30)

### 3. Валидация времени
- Проверять, что выбранное время валидно
- Бэкенд вернет ошибку при невалидном формате
- Показывать ошибку пользователю при некорректном времени

### 4. Отображение времени в списке заказов
- Показывать `scheduledAt` в карточке заказа
- Форматировать дату/время в читаемый формат
- Отображать статус просроченности через поле `isOverdue`

### 5. Тестирование
- Проверить создание заказа без указания времени (должно автоматически установиться +1 час)
- Проверить создание заказа на конкретное время (например, 11:00, даже если сейчас 16:30)
- Проверить отображение времени в созданном заказе
